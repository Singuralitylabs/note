# はじめてのターミナル — 黒い画面でのファイル操作を初心者向けにやさしく解説

## はじめに

前回の記事では、コードエディタ「VSCode」の画面構成や基本操作を紹介しました。エディタが使えるようになると、次に出てくるのが **ターミナル** という「黒い画面」です。

ターミナルは、マウスでクリックする代わりに、文字でコンピュータに命令を出すための道具です。最初は無機質で怖く見えるかもしれませんが、覚えるコマンドはほんの少しで十分です。そして一度慣れてしまうと、フォルダの移動やファイルの整理が、マウス操作よりずっと速くこなせるようになります。

この記事では、ターミナルを初めて触る人が最初に押さえておきたい基本だけに絞って解説します。難しい話は思い切って省き、「まずはここだけ」という土台づくりに集中します。

- ターミナルとシェルの正体を知る
- パス（ファイルの住所）の感覚をつかむ
- 移動・確認の3コマンドを覚える
- ファイルを作る・コピーする・消す
- ファイルや文字列を検索する
- 操作を速くするコツを知る

## ターミナルとシェルとは

「ターミナル」「シェル」「プロンプト」という言葉は一緒に使われがちですが、役割が少しずつ違います。たとえるなら、ターミナルは文字を入力・表示する **画面（アプリ）** そのもの、シェルはその入力を受け取って **解釈・実行するプログラム**、プロンプトは入力待ちを示す **記号**（**$** や **%**）です。

つまり、あなたがターミナルに文字を打つと、シェルがそれを読み取ってコンピュータに実行させ、終わるとまたプロンプトが出て次の入力を待つ、という流れになります。

シェルにはいくつか種類があり、macOSの標準は **zsh**、Linuxでは **bash** がよく使われます。基本的なコマンドはどちらも共通なので、最初は違いを気にせず進めて大丈夫です。

Windowsをお使いの方は、標準のPowerShellやコマンドプロンプトだとコマンドの挙動が変わってしまいます。Gitをインストールすると一緒に入る **Git Bash** を使うと、この記事のコマンドがそのまま使えるので、そちらをおすすめします。

ターミナルに打ち込むコマンドは、「何をするか（コマンド）」「どう動かすか（オプション）」「何に対して行うか（引数）」の3つの部品でできています。たとえば次のような構造です。

```bash
ls -l Documents
# ls       = コマンド（一覧表示）
# -l       = オプション（詳細表示。- で始まる）
# Documents = 引数（対象のフォルダ）
```

オプションや引数は省略できることも多く、**ls** だけでも動きます。

## パスの感覚をつかむ

ターミナルでは「どのファイルを？」「どこへ移動する？」をすべて **パス** で指定します。パスとは、ファイルやフォルダの「住所」を表す文字列のことです。

コンピュータの中のファイルは、一番上の **ルート（スラッシュ /）** を頂点とした入れ子構造で管理されています。たとえば **index.html** という1つのファイルを、ルートからたどって **/Users/taro/projects/my-app/index.html** のように書いたものが **絶対パス** です。どこにいても必ず同じ場所を指します。

一方、いまいる場所（これを **カレントディレクトリ＝現在地** と呼びます）を基準に書いたものが **相対パス** です。現在地が **/Users/taro** なら、**projects/my-app/index.html** だけで上と同じファイルを指せます。短く書ける反面、現在地が変わると指す場所も変わる点に注意しましょう。

パスではいくつかの記号がよく登場します。区切りやルートを表す **/**、ホームディレクトリ（自分の基地となるフォルダ）を表す **~**、現在地そのものを表す **.（ドット1つ）**、1つ上の親フォルダを表す **..（ドット2つ）** の4つを覚えておけば十分です。

## 移動と確認の3コマンド

ターミナル操作の土台は「現在地を確認する → 中身を見る → 移動する」という3つの動作です。これに対応するのが **pwd**・**ls**・**cd** の3コマンドです。

```bash
pwd          # 現在地（カレントディレクトリ）を表示
ls           # 現在地のファイル・フォルダ一覧を表示
cd projects  # projects フォルダへ移動
```

**ls** にはよく使うオプションがあります。**ls -l** で詳細表示、**ls -a** で隠しファイル（ドットで始まるもの）も表示、まとめて **ls -la** とも書けます。**.zshrc** のような設定ファイルは隠しファイルなので、**ls** だけでは見えません。「無いはずがないのに見えない」と思ったら **-a** を付けてみてください。

**cd** もいくつかの書き方を覚えておくと便利です。

```bash
cd ..        # 1つ上のフォルダへ
cd ~         # ホームディレクトリへ
cd           # 引数なしでもホームへ
cd -         # 直前にいたフォルダへ戻る
```

特に **cd -** は、2つのフォルダを行ったり来たりするときに重宝します。フォルダ名は大文字・小文字が区別されるので、打ち間違いを防ぐためにも、後で紹介する **タブ補完** を使うのがおすすめです。

## ファイルを作る・コピーする・消す

移動できるようになったら、次はファイルやフォルダを操作します。GUIでいう「新規作成・コピー・名前変更・ゴミ箱へ」にあたる操作です。

```bash
mkdir my-app          # フォルダを作る
touch index.html      # 空のファイルを作る
cp index.html bak.html # ファイルをコピーする
mv old.txt new.txt    # 移動する／名前を変更する
rm temp.txt           # 削除する
```

フォルダをまとめて扱うときは **-r**（中身も含めて、という意味）が必要です。たとえばフォルダごとコピーするなら **cp -r src src-backup**、フォルダごと削除するなら **rm -r old-folder** と書きます。

ここでひとつ、初心者がいちばん気をつけたいのが **rm** による削除です。

> ターミナルの **rm** で消したファイルは、ゴミ箱に入りません。すぐに完全に消え、GUIのように「ゴミ箱から戻す」ことはできません。

特に **rm -rf フォルダ名** は、確認なしで一気に削除してしまう強力なコマンドです。実行する前に、必ず **pwd** で現在地を、**ls** で対象を確認する習慣をつけましょう。不安なときは、1ファイルずつ確認しながら消せる **rm -i** を使うと安全です。

## ファイルを探す・中身を検索する

作業に慣れてくると、「あのファイルどこに置いたっけ？」「この設定はどのファイルに書いてある？」と探したくなる場面が増えます。このとき役立つのが **find** と **grep** という2つの検索コマンドです。

ふたつの違いはシンプルです。**find** は「○○という名前のファイルはどこ？」とファイル **そのもの** を名前で探すコマンド、**grep** は「○○という文字が書かれているのはどの行？」とファイルの **中身** を探すコマンドです。

まず **find** は、「どこから探すか」と「何を探すか」を続けて書きます。深い階層のどこかにあるファイルも一発で見つけられます。

```bash
find . -name "index.html"   # 現在地（.）以下から index.html を探す
find . -name "*.md"         # .md で終わるファイルをすべて探す
find . -type d              # フォルダだけを一覧する
```

最初の **.（ドット）** は「現在地から探す」という意味の検索開始地点です。**-name** で名前の条件を、**-type d** を付ければフォルダだけ、というように絞り込めます。名前に **\*** のような記号を使うときは、**"*.md"** のようにダブルクォートで囲むのが安全です。

次に **grep** は、ファイルの中身から、指定した文字を含む行を探し出します。コードを読むエンジニアが最もよく使うコマンドのひとつです。

```bash
grep "TODO" app.js       # app.js から TODO を含む行を探す
grep -n "TODO" app.js    # 行番号つきで表示する
grep -i "error" log.txt  # 大文字小文字を区別しない
grep -rn "useState" src/ # src フォルダ以下をまとめて検索（-r）+ 行番号
```

よく使うのは、行番号を表示する **-n**、大文字小文字を無視する **-i**、フォルダ全体をまとめて検索する **-r** の3つです。実務では「プロジェクト全体のどこで、この関数が使われている？」を調べるのに **grep -rn 関数名 src/** のような形を頻繁に使います。

「名前で探すなら **find**、中身で探すなら **grep**」と覚えておけば、まずは十分です。

## 操作を速く・ラクにするコツ

最後に、ターミナルに慣れた人が必ず使っているテクニックを紹介します。これを知っているかどうかで、作業スピードが大きく変わります。

ひとつ目は **タブ補完** です。フォルダ名やファイル名を途中まで打って **Tab** キーを押すと、残りを自動で補完してくれます。長い名前を全部打つ必要がなく、しかも実在する名前しか補完されないので、打ち間違いも防げます。

ふたつ目は **コマンド履歴** です。過去に打ったコマンドは **↑** キーで呼び出せます。さっき打った長いコマンドをもう一度実行したいときは、**↑** を押して **Enter** するだけです。**Ctrl + R** を押すと、キーワードで履歴を検索することもできます。

そして、覚えておくと安心なのが **Ctrl + C** です。コマンドが終わらない、画面が固まったと感じたら、まずこれで中断を試してください。ファイルの中身を **less** などで閲覧していて画面から戻れなくなったときは、**q** キーで抜けられます。

ファイルの中身をサッと見たいときは、短いファイルなら全部表示する **cat**、長いファイルならスクロールできる **less** が便利です。コマンドの使い方を忘れたら、暗記しようとせず **man コマンド名** でマニュアルを開けば調べられます。プロのエンジニアでも、細かいオプションはその都度調べています。

## まとめ

この記事では、ターミナルを初めて触る人向けに、最低限の基本だけを紹介しました。

- ターミラルは文字で命令を出す道具で、シェルがそれを解釈して実行する
- ファイルの住所である **パス** には、絶対パスと相対パスの2種類がある
- **pwd・ls・cd** の3つで現在地の確認と移動ができる
- **mkdir・touch・cp・mv・rm** でファイルを操作でき、特に **rm** は要注意
- 名前で探すなら **find**、中身で探すなら **grep**
- **タブ補完** と **履歴**、そして **Ctrl + C** を知っておくと作業が一気にラクになる

ここまで使えれば、ターミナルでの基本的な作業は十分にこなせます。実際に手を動かして「現在地を確認 → 移動 → 中身を見る」のリズムを体に覚えさせるのが上達の近道です。慣れてきたら、コマンドを組み合わせてさらに強力に使う方法（リダイレクトやパイプなど）にも、順番に挑戦してみてください。

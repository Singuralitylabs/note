# Claude Codeのカスタムスキル（Skills）

## はじめに

前回の記事では、Claude Codeから外部サービスと連携するMCPについて解説しました。今回は、Claude Codeに「自分専用のワークフロー」を覚えさせて何度でも呼び出せるようにする、カスタムスキル（Skills）機能を紹介します。

「同じ指示を毎回コピペしている」「CLAUDE.mdに手順を書きすぎて肥大化してきた」「チームでレビュー観点を揃えたい」といった課題を、マークダウンファイル一枚で解決できる仕組みです。

## Skillsとは何か

Skillsは、繰り返し使うワークフローをマークダウンで定義し、スラッシュコマンドとして呼び出せる仕組みです。

たとえば「セキュリティ・パフォーマンス・可読性の3観点でレビューして」という指示を **SKILL.md** に書いておくと、**/review** で何度でも呼び出せるようになります。

CLAUDE.mdが「常に適用されるルール」を書く場所だとすれば、Skillsは「必要なときだけ呼び出すワークフロー」を置く場所です。CLAUDE.mdに手順を書きすぎて重くなってきたら、Skillsに切り出すサインだと考えてください。

公式ドキュメントは[Anthropicの公式サイト](https://code.claude.com/docs/en/skills)で公開されています。仕様の詳細を確認したいときはこちらを参照してください。

## 既存のスキルを活用する

自分で作る前に、すぐに使えるスキルを覗いてみるのがおすすめです。

Claude Codeには **/simplify**（コード品質改善）や **/debug**（体系的なデバッグ支援）といったスキルが標準で組み込まれています。さらに、Anthropicが公開している公式スキル集（[anthropics/skills](https://github.com/anthropics/skills)）をプラグインとして追加することもできます。

```bash
# 公式スキル集を追加
/plugin install example-skills@anthropic-agent-skills
```

公式スキルには、対話形式でスキルを作ってくれる **/skill-creator** や、UI設計を支援する **/frontend-design** などがあります。

### コミュニティスキルを探す：skills.sh

Vercelが運営する[skills.sh](https://skills.sh)は、Claude Codeを含む複数のAIエージェント向けのスキルが集まるオープンディレクトリです。インストール数のランキングから人気スキルを発見でき、CLIから手軽に導入できます。

```bash
# skills.shのスキルをインストール
npx skills add <owner/repo>
```

中でもおすすめなのが、vercel-labsが公開している **find-skills** です。「○○ができるスキルはある？」と聞くだけで関連スキルを検索し、インストールコマンドまで提示してくれます。スキルを探すためのスキルという発想で、Skillsエコシステムへの入口に最適です。

## 最初のスキルを自作する

スキルはマークダウンファイル一枚で作れます。

### ステップ1：ディレクトリを作成する

プロジェクトでチーム共有するなら **.claude/skills/** 配下、自分専用なら **~/.claude/skills/** 配下にディレクトリを作ります。

```bash
# プロジェクト固有（チーム共有可能）
mkdir -p .claude/skills/review
```

### ステップ2：SKILL.mdを書く

作ったディレクトリの中に **SKILL.md** を置きます。先頭にフロントマターでメタ情報、その下に指示内容を書きます。

```markdown
---
name: review
description: コードレビューを実行する。レビュー依頼やPRのチェックを求められたときに使用。
---

# Code Review Skill

以下の観点でコードレビューを行ってください。

- セキュリティ：SQLインジェクション、XSS、認証漏れ
- パフォーマンス：N+1クエリ、不要な再レンダリング
- 可読性：命名規則、関数の長さ
- テスト：カバレッジ、エッジケースの網羅

重要度（高・中・低）と具体的な修正案を必ず含めてください。
```

**name** がスラッシュコマンド名になります。**description** はClaude Codeが自動でスキルを使うべきか判定するときに参照されるため、「何をするか」「どんなときに使うか」を具体的に書くと精度が上がります。

### ステップ3：呼び出す

保存したらすぐ使えます。セッションの再起動は不要です。

```bash
/review
```

## 呼び出し方は2パターン

スキルは2通りの方法で呼び出されます。

**手動呼び出し**は、**/review** のようにスラッシュコマンドで明示的に実行する方法です。

**自動呼び出し**は、会話の文脈に応じてClaudeが必要なスキルを自動で読み込む仕組みです。「このコードレビューして」と頼むだけで、裏で **/review** スキルが動くイメージです。

自動呼び出しの精度は **description** の書き方で決まります。抽象的な記述だと検出されにくいので、用途と状況を具体的に書くのがコツです。

## まとめ

カスタムスキルを使いこなすと、普段の指示は再利用可能なコマンドになり、チーム全体で同じワークフローを共有できます。

- Skillsは「必要なときだけ呼び出す」ワークフロー
- CLAUDE.mdの肥大化を防ぎ、コンテキストを圧迫しない
- マークダウン一枚で自作でき、すぐにスラッシュコマンドとして使える

まずは普段繰り返している指示を一つ、SKILL.mdに切り出すところから始めてみてください。

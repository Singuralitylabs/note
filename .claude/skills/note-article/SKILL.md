# note記事作成スキル

noteで公開するWeb系技術記事のドラフトを作成するためのワークフローです。

## 配置場所と命名規則

記事はトピックに応じてカテゴリディレクトリに配置します。

- AI関連：`ai_development/`
- Web技術：`web_development/`
- コラム：`columns/`
- 業務関連：`work/`

ファイル名は `NN_topic-slug.md` の形式（例：`08_claude-code_skills.md`）。同じカテゴリの既存ファイルから連番を引き継ぐ。

## 必ず守る記事構造

```
# 記事タイトル（h1は記事冒頭の1回のみ）

## はじめに
（前回記事の振り返り → 今回扱うテーマの導入）

## トピック1
## トピック2
## トピック3
...

## まとめ
（要点の再掲 → 読者への次のアクション提案）

---

## プログラミングイベントのご案内
...（定型フッター3セクション）
```

末尾の区切り線（`---`）はフッター直前のみで使用する。本文中のセクション間には絶対に使わない。

## noteで使用可能なマークダウン

- 見出し：`##`（h2）と `###`（h3）のみ。h1は記事タイトルの1箇所、h4以降は使用不可
- 強調：`**太字**`、`~~取り消し線~~`
- 引用：`> `
- 区切り線：`---`（フッター直前のみ）
- リンク：`[テキスト](URL)`
- コードブロック：` ``` ` で囲み、言語指定でシンタックスハイライト

## noteで使用してはいけない記法

以下は描画されないか変換されてしまうため絶対に使わない。

- テーブル（`|` を使った表）→ 文章または箇条書きに展開する
- インラインコード（`` `code` ``）→ `**太字**` に置き換える
- 画像埋め込み（`![alt](url)`）→ HTMLの `<figure>` + `<img>` で記述
- 番号付きリスト（`1. ` 形式）→ 「ステップ1：」のような見出しか文章に展開
- 箇条書きのネスト → フラットな箇条書きか文章に分解

## 文章スタイル

- 「です・ます」調で統一
- 専門用語は初出時に簡潔な補足を添える
- コード例は実用的で短いものに絞る
- 公式ドキュメントなど一次情報があればリンクで埋め込む（例：`[公式ドキュメント](https://...)`）
- 強調したいキーワードや設定値は **太字** で表現（インラインコードの代替）

## 作成手順

1. **配置先と連番を決める**：対象カテゴリのディレクトリを `ls` で確認し、最終番号+1を採用する
2. **前回記事の末尾を確認する**：シリーズものの場合は前回記事を読んで、「はじめに」で自然に繋げる
3. **構成案を立てる**：「はじめに → 主要トピック3〜6本 → まとめ」の骨格で、各セクションの主旨を一文で書き出す
4. **本文を書く**：上記のスタイルと禁止記法を守って執筆
5. **フッターを貼り付ける**：定型フッターを末尾に追加
6. **禁止記法をセルフチェックする**：テーブル・インラインコード・番号付きリスト・ネスト箇条書きが残っていないか最終確認

## 分量の目安

- 入門解説記事：本文 1,500〜3,000字程度
- 主要セクションは3〜6本
- 「重要なところに絞って」と指示された場合は、概要・既存活用・自作・呼び出し方・まとめの5本程度に絞り込む
- 細部（全フィールド一覧、トラブルシューティング、発展機能の詳細など）は基本的に省略し、要望があれば追加する
